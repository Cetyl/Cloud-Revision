# Local AI Code Generator

A fully offline, private AI-powered code generator that runs 100% on your MacBook. No ChatGPT. No internet. No API costs.

Built with Qwen2.5 (7B), Ollama, and Streamlit.

---

## What It Does

Type a plain English description of what you need. The local AI model processes it entirely on your machine and returns production-ready code instantly.

Supported output types:

- Terraform Code
- Python Scripts
- AWS CloudFormation YAML
- Dockerfiles
- CI/CD Pipelines (GitHub Actions)
- Bash / Shell Scripts
- SQL Queries
- General Code (any language)

---

## Tech Stack

| Layer | Tool |
|---|---|
| Model | Qwen2.5:7b (open source, fully offline) |
| Model Server | Ollama |
| UI | Streamlit |
| Language | Python |
| Hardware | MacBook with 32GB unified memory |
| Internet Required | None |

---

## Architecture

```
You (Browser at localhost:8501)
        ↓
Streamlit UI
        ↓
Ollama (local model server)
        ↓
Qwen2.5:7b (runs on your MacBook)
        ↓
Generated Code returned to UI
```

Everything runs locally. Zero data leaves your machine.

## Prerequisites

- MacBook with Apple Silicon (M1 / M2 / M3)
- 16GB RAM minimum (32GB recommended)
- Python 3.11+
- Homebrew installed

---

## Installation

**Step 1 — Install Ollama**
```bash
brew install ollama
```

**Step 2 — Download Qwen2.5 model** (approx 4GB)
```bash
ollama serve
ollama pull qwen2.5:7b
```

**Step 3 — Clone this repo**
```bash
git clone https://github.com/YOUR_USERNAME/local-ai-codegen.git
cd local-ai-codegen
```

**Step 4 — Create virtual environment and install dependencies**
```bash
python3 -m venv venv
source venv/bin/activate
pip install streamlit ollama
```

---

## Running the App

The easiest way — just run the start script:

```bash
bash start.sh
```

This will automatically:
- Create the virtual environment if it does not exist
- Install all dependencies
- Start Ollama
- Launch the Streamlit app at localhost:8501

---

## Manual Start

If you prefer to run it manually:

```bash
# Terminal 1 — Start Ollama
ollama serve

# Terminal 2 — Run the app
source venv/bin/activate
streamlit run app.py
```

Open your browser at `localhost:8501`

---

## Shutting Down

When you are done:

```bash
pkill ollama
```

**Python:**
```
Write a Python AWS cost calculator that takes
number of EC2 instances, instance type, storage
in GB and data transfer in GB and calculates
monthly and yearly cost
```

```
Create a VPC with two public and two private
subnets across two availability zones
with an internet gateway and NAT gateway
```

```
Dockerfile for a Python FastAPI app using
Python 3.11, installs requirements.txt,
runs on port 8000, uses non-root user
```

```
CI/CD pipeline that runs on push to main,
runs pytest, builds Docker image
and pushes to AWS ECR
```

## Live Troubleshooting

During testing, the generated AWS Cost Calculator had a bug — an instance type was missing from the price list. Instead of switching to ChatGPT, the error was fed back into the local AI with a more specific prompt. It self-corrected and returned clean working code.

That is the real value here — fast iteration loop, fully private, no data ever leaving your machine.

## Why This Matters

In real client engagements, you cannot always paste architecture details or infrastructure configurations into ChatGPT. Confidentiality is a real constraint. A local model that helps you generate code, think through infrastructure, and iterate privately is genuinely useful.
